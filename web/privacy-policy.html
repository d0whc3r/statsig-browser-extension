Privacy Policy

Last updated: March 31, 2026

This Privacy Policy explains how the Statsig Browser Extension handles information when you use the extension. The extension runs entirely in your browser and does not collect, store, or transmit personal data to any third-party server.

Information we process

  • Extension settings you configure, such as your Statsig Console API Key and storage preferences.
  • Statsig evaluation data (gates, experiments, dynamic configs) displayed in the extension interface, fetched directly from the Statsig API using your credentials.
  • Temporary runtime information required to render the extension UI, including the detected Statsig user object from the current page.

How information is used

  • To display Statsig entities and their current evaluation state in the extension popup.
  • To allow authorized override actions initiated by you through the Statsig Console API.
  • To support debugging and QA workflows without leaving your application.
  • All data is stored locally in your browser (localStorage or cookies, per your preference).

Data sharing

The extension does not sell, share, or transmit personal data to any party other than the Statsig API endpoints required for its functionality. API requests are made only when you trigger an action (e.g., applying an override) and use the API Key you provide.

Security

We use reasonable measures to protect data handled by the extension. Your API Key is stored locally in your browser and is never transmitted to any server other than the official Statsig API. You are responsible for safeguarding your own credentials and access keys.

Your choices

You can clear all extension data at any time by clearing your browser's extension storage or uninstalling the extension. You can also change your storage preference (localStorage vs cookies) in the extension settings.

Contact

For questions about this policy, please reach out via the GitHub repository issues.
